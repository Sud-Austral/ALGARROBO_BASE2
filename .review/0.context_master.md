# 📘 CONTEXTO MAESTRO: ALGARROBO_BASE2 (Geoportal Municipal)

## 📋 METADATA
- **Fecha**: 2026-03-30
- **Stack**: Vanilla JS (ES6+), HTML5, CSS3 (Tailwind CDN / DaisyUI), Python Flask (Backend)
- **Browser Support**: Modern only (Fetch API, ES6 modules, Flexbox/Grid)
- **Criticidad**: ALTA (Sistema de gestión gubernamental/municipal con datos sensibles)

## 🎯 OBJETIVOS DE NEGOCIO
Proporcionar una plataforma centralizada para el seguimiento de proyectos municipales, gestión de licitaciones, transparencia y auditoría, integrando herramientas de IA para análisis de datos y consulta ciudadana.

## 🏗️ ARQUITECTURA TÉCNICA
- **Patrón**: Multi-Page Application (MPA) con lógica compartida mediante scripts globales (`layout.js`, `router.js`, `api.js`).
- **Flujo de Datos**: 
    - **Frontend -> Backend**: Fetch API nativa centralizada en `api.js` con manejo de Bearer Tokens en LocalStorage.
    - **Backend -> Frontend**: JSON REST API.
    - **UI**: Manipulación directa del DOM (`document.getElementById`, `.innerHTML` con sanitización).
- **Web Standards Compliance**: Alto uso de Web APIs nativas. No se detectan bundlers (Webpack/Vite) ni transpilers (Babel). Dependencia crítica de CDNs externos (FontAwesome, Tailwind, DOMPurify).
- **Restricciones de Entorno**: 
    - Despliegue en Railway.
    - Uso de PostgreSQL (NeonDB).
    - CORS restrictivo basado en variables de entorno (`ALLOWED_ORIGINS`).

## ⚠️ RESTRICCIONES Y ÁREAS SENSIBLES
- **Archivos No Tocar (Lógica de Seguridad)**:
    - `frontend/script/router.js`: Control de acceso y redirección por roles.
    - `frontend/script/layout.js`: Inyección dinámica de UI y cierre de sesiones.
- **Deuda técnica identificada**:
    - Fuerte acoplamiento a CDNs externos sin fallback local.
    - Mezcla de lógica de negocio y manipulación de DOM en archivos `.html`.
    - Uso de `innerHTML` (aunque mitigado con script de sanitización).
