# 🏗️ REPORTE DE ARQUITECTURA: ALGARROBO_BASE2

## 🎯 ANÁLISIS MACRO
- **Patrón Detectado**: Multi-Page Application (MPA) con inyección dinámica de JavaScript nativo.
- **Acoplamiento**: ALTO (Cada página HTML repite lógica de negocio similar).
- **Escalabilidad**: MEDIA-BAJA (Dificultad para añadir nuevas secciones sin duplicar código).

---

**🏗️ Aspecto**: Gestión de UI y Plantillas (Templates)
**📐 Patrón Detectado**: String Interpolation / Direct Injection.
**⚠️ Riesgo Arquitectónico**: Frágil y difícil de testear. Un error en un string detiene el renderizado total. Carga pesada para el navegador al re-parsar el DOM constantemente.
**📈 Recomendación**: Migrar hacia **Native Web Components** o un sistema de plantillas `<template>` reutilizables en archivos externos importados vía Fetch o ESM.

---

**🏗️ Aspecto**: Comunicación entre Componentes
**📐 Patrón Detectado**: Singleton Global (`api.js`, `utils.js`).
**⚠️ Riesgo Arquitectónico**: Dependencia oculta al cargar scripts en orden. Si `api.js` falla, el resto de componentes lanzan errores de referencia.
**📈 Recomendación**: Implementar un **Event Bus nativo** (`window.dispatchEvent`) o usar el patrón **Mediator** para que el dashboard y el sidebar se comuniquen sin conocerse directamente.

---

**🏗️ Aspecto**: Gestión de Sesión y Seguridad (Router)
**📐 Patrón Detectado**: Interceptor Global en `router.js`.
**⚠️ Riesgo Arquitectónico**: Correcto, pero excesivamente dependiente del LocalStorage. Si el LocalStorage es manipulado (aunque el token sea válido), la UI podría renderizar datos inconsistentes antes de la verificación del backend.
**📈 Recomendación**: Implementar un **State Store centralizado** (Proxy pattern) que sea la única fuente de verdad para el estado de login y roles, validando contra el backend de forma más íntegra.

---

**🏗️ Aspecto**: Distribución de Código
**📐 Patrón Detectado**: Flat structure con scripts compartidos.
**⚠️ Riesgo Arquitectónico**: Difusión de lógica de negocio (Business Logic) en las capas de presentación (HTML).
**📈 Recomendación**: Mover la lógica de los archivos `.html` a archivos `.js` dedicados por página o módulo, separando la lógica de eventos de la lógica de renderizado.
