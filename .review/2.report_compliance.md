# 🛡️ REPORTE DE CUMPLIMIENTO (COMPLIANCE): ALGARROBO_BASE2

## 🎯 RESUMEN EJECUTIVO
- **Estándar**: OWASP Top 10 / Supply Chain Security.
- **Riesgos Identificados**: 5 críticos.
- **Estado de SRI**: INCOMPLETO (Detectado uso de CDNs sin integridad).

---

**❗ Hallazgo**: Exposición de secretos en el cliente (Hardcoded API Key).
**🔓 Vulnerabilidad**: Secret Exposure.
**📍 Ubicación**: `frontend/division/secplan/admin_general/chat.html:493`
**⚠️ Severidad**: CRÍTICA.
**📜 Estándar**: OWASP A07:2021 – Identification and Authentication Failures.
**🛠️ Mitigación**: La función `getKey` realiza una ofuscación débil (XOR/Base64) que es fácilmente reversible. Se debe mover la lógica de integración con ZhipuAI al backend (Flask) para que la API Key nunca viaje al navegador del usuario.

---

**❗ Hallazgo**: Uso de CDNs sin Subresource Integrity (SRI).
**🔓 Vulnerabilidad**: Supply Chain / Script Injection.
**📍 Ubicación**: `index.html` (Tailwind, FontAwesome), `chat.html` (Marked, Tailwind).
**⚠️ Severidad**: ALTA.
**📜 Estándar**: OWASP A08:2021 – Software and Data Integrity Failures.
**🛠️ Mitigación**: Aunque existe un script automatizado para inyectar SRI, se detectan archivos `.html` que aún cargan librerías críticas como Tailwind o FontAwesome sin los atributos `integrity` y `crossorigin`. Es imperativo asegurar que el script `refactor_backend.js` se ejecute en cada deploy o manualmente en todos los archivos.

---

**❗ Hallazgo**: Almacenamiento de PII en LocalStorage.
**🔓 Vulnerabilidad**: Data Leak / Insecure Storage.
**📍 Ubicación**: `frontend/script/router.js:90`, `index.html:294`
**💥 Escenario**: Se guarda el objeto `userData` completo (que incluye nombres y roles) en `localStorage`. Cualquier script malicioso (XSS) ejecutado en el mismo origen podría exfiltrar esta información.
**⚠️ Severidad**: MEDIA.
**📜 Estándar**: GDPR / OWASP A04:2021 – Insecure Design.
**🛠️ Mitigación**: Almacenar únicamente el JWT. La información del usuario debe ser recuperada mediante un endpoint `/auth/me` con el token, o almacenarse de forma temporal en un `state` de JS (SessionStorage) que desaparezca al cerrar la pestaña.

---

**❗ Hallazgo**: Falta de Content Security Policy (CSP).
**🔓 Vulnerabilidad**: XSS / Data Exfiltration.
**📍 Ubicación**: Headers del servidor (Flask) / Meta Tags.
**⚠️ Severidad**: ALTA.
**📜 Estándar**: OWASP A03:2021 – Injection.
**🛠️ Mitigación**: El backend no envía cabeceras CSP. Esto permite que el frontend conecte con orígenes no autorizados o ejecute inline-scripts. Se debe implementar una política estricta que solo permita conexiones a la API propia y CDNs confiables con SRI.
